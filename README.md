# Классификатор обращений для автоматизации службы поддержки

## Описание проекта

Проект представляет собой ML-сервис для автоматической классификации текстовых обращений в службу поддержки.

Сервис принимает текст обращения клиента и возвращает:

- категорию обращения;
- приоритет обработки.

Пример входных данных:

```json
{
  "text": "У меня не проходит оплата картой"
}
```

Пример ответа:

```json
{
  "category": "Оплата",
  "priority": "Высокий"
}
```

## Цель проекта

Цель проекта — разработать программный модуль, который помогает автоматизировать первую линию службы поддержки: сортирует обращения клиентов по категориям и определяет приоритет обработки.

## Категории обращений

В проекте используются следующие категории:

- Оплата
- Техническая ошибка
- Доставка
- Возврат и обмен
- Спам

## Датасет

После очистки датасет содержит 720 уникальных обращений.

Проверка качества данных:

- пропуски в тексте: 0;
- полные дубли: 0;
- дубли по тексту: 0.

Распределение категорий:

| Категория | Количество |
|---|---:|
| Оплата | 194 |
| Доставка | 160 |
| Техническая ошибка | 157 |
| Возврат и обмен | 114 |
| Спам | 95 |

## Приоритеты

Приоритет определяется по предсказанной категории:

| Категория | Приоритет |
|---|---|
| Оплата | Высокий |
| Техническая ошибка | Высокий |
| Доставка | Средний |

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Komi-Mir/Request-classifier-for-support-service-automation.git
cd Request-classifier-for-support-service-automation
```

### 2. Установить зависимости

```bash
python -m pip install -r requirements.txt
```

### 3. Обучить модель

```bash
python model.py
```

После обучения создаются файлы:

- `model.pkl`
- `TfIdfVectorizer.pkl`
- `label_encoder.pkl`

### 4. Запустить API

```bash
python app.py
```

API запускается локально по адресу:

```txt
http://127.0.0.1:8000
```

Основной endpoint:

```txt
POST /predict
```

Пример запроса через PowerShell:

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/predict" `
  -Method POST `
  -ContentType "application/json; charset=utf-8" `
  -Body '{"text":"У меня не проходит оплата картой"}'
```

Пример ответа:

```txt
category priority
-------- --------
Оплата   Высокий
```

### 5. Запустить интерфейс

В отдельном терминале:

```bash
streamlit run streamlit_app.py
```

После запуска откроется веб-интерфейс, где можно ввести текст обращения и получить категорию и приоритет.

## Метрики модели

Финальная модель:

- алгоритм: TF-IDF + Logistic Regression
- Accuracy: `0.9236`
- F1-score weighted: `0.9223`
